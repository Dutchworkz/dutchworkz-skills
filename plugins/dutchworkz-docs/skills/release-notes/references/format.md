# Release notes — volledig format

Dit bestand wordt alleen geladen wanneer de agent het nodig heeft (progressive disclosure).

## Template

```markdown
# Release <versie> — <datum in formaat: 11 juni 2026>

> Eén zin die de release samenvat voor iemand die alleen de eerste regel leest.

## 🚀 Features
- <wijziging vanuit gebruikersperspectief> (#<pr>)

## 🐛 Fixes
- <wijziging> (#<pr>)

## ⚠️ Breaking changes
- <wijziging> — **Actie vereist:** <wat moet de gebruiker doen>

## 🔧 Overig
- <wijziging>

---
*Vragen over deze release? Meld je in het #releases-kanaal.*
```

## Versienummering

- Volg SemVer: MAJOR bij breaking changes, MINOR bij features, PATCH bij alleen fixes.
- Stel op basis van de gegroepeerde wijzigingen een versienummer voor als de gebruiker er geen noemt.

## Toon

- Zakelijk maar menselijk; geen jargon dat een productowner niet begrijpt.
- Geen superlatieven ("geweldige nieuwe feature"), wél concreet resultaat ("rapportage exporteert nu naar Excel").
